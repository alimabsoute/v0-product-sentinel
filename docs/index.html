Foundation — Scripts & Migrations to Build

TASKS

  1. 1

    DB Migration 0008 — github_snapshots table

    New table: product_id, snapshot_date, stars, forks, open_issues. Daily snapshot for velocity calculation.

    new table ~10 min
  2. 2

    DB Migration 0009 — signal score github col

    ALTER product_signal_scores ADD github_velocity_score NUMERIC(5,2) DEFAULT 0.

    alter table~5 min
  3. 3

    DB Migration 0010 — attribute_cohort_share view

    SQL view joining product_tags × products.launched_year → share_pct per tag per cohort. Powers the attribute market share chart.

    new view~15 min
  4. 4

    scripts/snapshot-github-stars.ts

    Reads products WHERE github_repo IS NOT NULL. Calls GET /repos/{owner}/{repo} via GitHub REST API. Upserts into github_snapshots. 1.2s delay between calls. GITHUB_TOKEN optional (60→5000 req/hr).

    new script~4h build$0 cost
  5. 5

    scripts/enrich-attributes.ts (most important)

    Firecrawl scrapes homepage + /pricing + /features per product. Haiku classifies into 10 tag groups. Inserts into product_tags. Skip-if-already-tagged for safe re-runs.

    new script~1 day build~$50-70 one-time run
  6. 6

    Modify compute-signal-scores.ts

    Add fetchGitHubVelocity() function. Add computeGitHubStarVelocityScore() (log scale, 0-15 pts). Move cron from 3am → 5am (after github snapshot at 4am).

    modify existing~2h

ONGOING MONTHLY

ItemCost/moFrequency
GitHub star snapshots $0 Daily · GitHub free tier
enrich-attributes re-run (new products) ~$2-5 Weekly · only new/untagged products
Signal score recompute $0 Daily · pure SQL
Ingest pipelines (HN, Reddit, News, PH) $0 Free APIs
Supabase (Pro if needed) $25 Only if >500MB storage or row limits hit
