4-Layer Attribute Taxonomy

The "prism" concept — breaking every product into its component facets. This is what makes every unbuilt feature possible.

Layer 1 — Hierarchical (single-value per product)
category~15 top-level · "AI Tools", "Dev Tools"
sub_category~100 mid-level · "Code Editors", "CRM"
primary_function~400 leaf values · "ai-code-completion"
Leaf is FK to functions table. Category + sub-category auto-derive from parent chain.
Layer 2 — Multi-select (product_tags JOIN tags)
capabilitycollaborative, real-time, ai-assist, api-first…
audiencedevelopers, enterprise, solopreneurs…
deploymentcloud, self-hosted, desktop-app, cli…
pricing_modelfree, freemium, open-source, paid-only…
tech_stackreact, rust, python, go, typescript…
ux_patternkeyboard-first, no-code, drag-drop, kanban…
integrationslack, notion, zapier, github, figma…
compliancesoc2, hipaa, gdpr, ccpa, iso27001…
data_formatmarkdown, json, csv, sql, images, video…
business_modelsaas, marketplace, open-source-funded…
Layer 3 — Task Discovery (TEXT[])
"write cold emails"
"summarize meeting notes"
"generate SQL queries"
"remove image backgrounds"
"transcribe video"
Powers /tools-to/[task] SEO pages. Massive long-tail traffic driver.
Layer 4 — Dimensional Scoring (JSONB)
